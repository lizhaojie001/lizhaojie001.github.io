Git的简单使用

前言

一些使用小技巧,为了以后的方便,特此记下

使用方法

环境:本地项目开始未配置git ,后来打算上传至github

未创建本地数据库

这种情况就比较好办了 首先,创建一个远程仓库,checkout到本地,然后将你的项目拖到当前目录下面,此时不要急着去commit,当然xcode的也是不许你这么做的,因为你的项目还未添加到本地仓库,在checkout的时候实际上已经创建了本地仓库,你只要做的是add进去你的项目就行了,在终端切换至当前项目 运行git add . 命令, 此时,你再去xcode上提交时,?已经全部消失,变为了M,此时你就可以进行提交了,让后push到远程仓库.

创建了本地数据库或者远程数据库

这种情况稍微有一点的麻烦,你一心想着上传至远程,但是,总是报错显示有些改变未提交,在检查提交后,并没什么没提交,但是就是不能上传,此时你就必须删除你项目中的本地库(删除本地和远程的仓库),也就是删除你项目中的.git文件,当然你的之前的提交记录也会被删除,这也要注意.一般.git文件都是隐藏的,我是利用终端去打开.git文件,然后将其删除,之后按照上面的方法一操作即可
